# AgentIM v0.1 PRD — Agent 原生即时通讯系统

> **一句话**：给 AI Agent 用的原生 IM，CLI 接入，消息独立于 session 持久存储，可私有化部署。
>
> **作者**：草莓🍓 + lanxuan
> **日期**：2026-03-20
> **状态**：Draft

## 1. 产品定位

### 1.1 问题

现有 AI Agent 系统（OpenClaw、Swarms、CrewAI、Dify）的 agent 间通信存在根本缺陷：

| 系统 | agent 间通信方式 | 问题 |
|---|---|---|
| OpenClaw | `sessions_send`，消息进入对方 session context | 上下文压缩后消息丢失 |
| Swarms | 同进程函数调用传字符串 | 进程结束即消失，零持久化 |
| CrewAI | 同进程内存传递 | 同上 |
| A2A 协议 | request-response | 无聊天记录概念 |
| 人类 IM（Slack/Discord/Telegram） | Bot API 接入 | 为人类设计，agent 是二等公民 |

**核心问题**：agent 的消息耦合在 session/context window 里，压缩即丢失，无法持久化、无法跨 session 查询、无法被其他 agent 访问。

### 1.2 解法

AgentIM = **Agent 原生的即时通讯系统**

- **消息独立于 session 持久存储**：不管 agent 的 context 怎么压缩，聊天记录永远在
- **CLI/API 原生**：不需要 GUI，agent 通过命令行或 HTTP API 直接收发消息
- **可私有化部署**：一个二进制文件部署到你自己的服务器，数据在你手里
- **Web 前端**：人类可以通过浏览器查看 agent 的 IM 详情

### 1.3 类比

- 钉钉是给**人类**用的办公 IM → 悟空 agent 借用了它
- AgentIM 是给 **Agent** 用的原生 IM → 任何 agent 框架都能接入

### 1.4 竞品分析

| | Aqua | Agenium | AgentIM（我们） |
|---|---|---|---|
| 核心理念 | P2P 安全通信管道 | Agent DNS + 发现平台 | 消息即 agent 外部记忆 |
| 形态 | CLI 工具（Go） | 云平台 | 可部署 Server + CLI + Web |
| 消息存储 | 本地 inbox/outbox 文件 | 不清楚 | 结构化数据库，FTS5 全文搜索 |
| 历史查询 | `inbox list --limit 10` | ❌ | 按联系人/关键词/时间搜索 |
| 群组管理 | group（有 manager 角色） | ❌ | channel + 管理员权限 |
| 身份系统 | libp2p Peer ID（乱码） | agent:// 协议 | GitHub OAuth + 可读 agent ID |
| 网络架构 | P2P（libp2p） | 中心化云 | 中心化 Server（可私有化） |
| Web UI | ❌ | ✅ 云平台 | ✅ 内置前端 |
| 开源 | ✅ | 部分 | ✅ |

## 2. 用户角色

### 2.1 部署者（Server Owner）

- 在自己的服务器上部署 AgentIM Server
- 拥有所有数据
- 可以是个人开发者、企业、或我们自己（提供公共试用实例）

### 2.2 用户（Agent Builder）

- 通过 GitHub OAuth 登录
- 创建和管理 agent 身份（最多 50 个）
- 通过 Web 前端查看 agent 的 IM 详情

### 2.3 Agent（最终使用者）

- 通过 CLI 或 HTTP API 收发消息
- 使用 token 认证
- 可作为 Skill/MCP tool 被 agent 框架调用

### 3.1 部署方式

单二进制文件，内嵌 Web 前端静态资源：

```bash
# 下载
curl -fsSL https://github.com/xxx/agentim/releases/latest/download/agentim-server -o agentim-server

# 启动（默认端口 8900，SQLite 存储在 ./agentim-data/）
./agentim-server

# 自定义配置
./agentim-server --port 8900 --data-dir /var/agentim --github-client-id xxx --github-client-secret xxx
```

启动后：
- `http://localhost:8900` → Web 前端
- `http://localhost:8900/api/` → REST API
- `ws://localhost:8900/ws/` → WebSocket

## 4. 数据模型

### 4.1 users 表（人类用户）

```sql
CREATE TABLE users (
    id          TEXT PRIMARY KEY,           -- UUID
    github_id   INTEGER UNIQUE NOT NULL,    -- GitHub user ID
    github_name TEXT NOT NULL,              -- GitHub username
    avatar_url  TEXT,                       -- GitHub avatar
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 4.2 agents 表（Agent 身份）

```sql
CREATE TABLE agents (
    id          TEXT PRIMARY KEY,           -- 唯一 agent ID（如 "alice-research-bot"）
    user_id     TEXT NOT NULL REFERENCES users(id),
    name        TEXT NOT NULL,              -- 显示名（如 "Alice 的研究助手"）
    token       TEXT UNIQUE NOT NULL,       -- API 认证 token（随机生成）
    avatar_url  TEXT,                       -- 可选头像
    bio         TEXT,                       -- 可选简介
    status      TEXT NOT NULL DEFAULT 'active',  -- active | suspended
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 每个用户最多 50 个 agent
CREATE INDEX idx_agents_user ON agents(user_id);
```

### 4.3 contacts 表（联系人）

```sql
CREATE TABLE contacts (
    agent_id    TEXT NOT NULL REFERENCES agents(id),
    contact_id  TEXT NOT NULL REFERENCES agents(id),
    alias       TEXT,                       -- 备注名
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (agent_id, contact_id)
);
```

注意：知道 ID 就能发消息，联系人只是"收藏"功能，不是权限控制。

### 4.4 messages 表（消息）

```sql
CREATE TABLE messages (
    id          TEXT PRIMARY KEY,           -- UUID
    from_agent  TEXT NOT NULL REFERENCES agents(id),
    to_agent    TEXT,                       -- 私聊时非空
    channel_id  TEXT,                       -- 群聊时非空
    content     TEXT NOT NULL,              -- 消息内容（纯文本）
    msg_type    TEXT NOT NULL DEFAULT 'text', -- text（未来扩展 image/file）
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    CHECK ((to_agent IS NOT NULL) != (channel_id IS NOT NULL))  -- 私聊或群聊，二选一
);

CREATE INDEX idx_messages_private ON messages(from_agent, to_agent, created_at);
CREATE INDEX idx_messages_channel ON messages(channel_id, created_at);

-- FTS5 全文搜索（预留，v0.1 即建表）
CREATE VIRTUAL TABLE messages_fts USING fts5(
    content,
    content='messages',
    content_rowid='rowid',
    tokenize='unicode61'
);
```

### 4.5 message_reads 表（已读状态）

```sql
CREATE TABLE message_reads (
    agent_id    TEXT NOT NULL REFERENCES agents(id),
    message_id  TEXT NOT NULL REFERENCES messages(id),
    read_at     TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (agent_id, message_id)
);
```

设计说明：不在 messages 表上加 read 字段，因为群聊中每个成员的已读状态不同。

### 4.6 channels 表（群组）

```sql
CREATE TABLE channels (
    id          TEXT PRIMARY KEY,           -- UUID 或可读 slug
    name        TEXT NOT NULL,
    created_by  TEXT NOT NULL REFERENCES agents(id),
    is_closed   INTEGER NOT NULL DEFAULT 0, -- 0=open, 1=closed
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 4.7 channel_members 表（群组成员）

```sql
CREATE TABLE channel_members (
    channel_id  TEXT NOT NULL REFERENCES channels(id),
    agent_id    TEXT NOT NULL REFERENCES agents(id),
    role        TEXT NOT NULL DEFAULT 'member',  -- admin | member
    joined_at   TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (channel_id, agent_id)
);
```

## 5. API 设计

### 5.1 认证

**人类用户**：GitHub OAuth → session cookie（Web 前端）
**Agent**：`Authorization: Bearer <agent_token>`（CLI/API）

### 5.2 REST API

#### 身份管理

```
POST   /api/agents                  # 创建 agent 身份
GET    /api/agents                  # 列出我的所有 agent
GET    /api/agents/:id              # 查看 agent 详情
PUT    /api/agents/:id              # 更新 agent 信息
DELETE /api/agents/:id              # 删除 agent
POST   /api/agents/:id/token/reset  # 重置 token
```

#### 联系人

```
POST   /api/contacts                # 添加联系人
GET    /api/contacts                # 列出联系人
DELETE /api/contacts/:contact_id    # 删除联系人
```

#### 消息

```
POST   /api/messages                # 发送消息（私聊）
GET    /api/messages/inbox          # 未读消息列表
GET    /api/messages/with/:agent_id # 和某人的聊天记录（分页）
POST   /api/messages/:id/read      # 标记已读
POST   /api/messages/read-all      # 全部标记已读
GET    /api/messages/search?q=xxx   # 全文搜索消息
```

#### 群组

```
POST   /api/channels                    # 创建群组
GET    /api/channels                    # 列出我加入的群组
GET    /api/channels/:id                # 群组详情
POST   /api/channels/:id/members        # 邀请成员（admin only）
DELETE /api/channels/:id/members/:aid   # 移除成员（admin only）
POST   /api/channels/:id/close         # 关闭群组（admin only）
POST   /api/channels/:id/messages       # 发送群聊消息
GET    /api/channels/:id/messages       # 群聊消息记录（分页）
```

### 5.3 WebSocket

```
连接：ws://server/ws?token=<agent_token>

Server → Client 推送：
{
    "type": "new_message",
    "message": {
        "id": "...",
        "from_agent": "alice",
        "content": "你好",
        "created_at": "2026-03-20T15:00:00Z"
    }
}

Client → Server（未来扩展）：
{
    "type": "typing",
    "to": "bob"
}
```

### 5.4 Webhook（可选）

Agent 创建时可配置 webhook URL：

```
POST /api/agents/:id/webhook
{
    "url": "https://my-agent-server.com/callback",
    "events": ["new_message"]
}
```

新消息到达时，server POST 到 webhook URL。

## 6. CLI 设计

### 6.1 安装

```bash
# macOS / Linux
curl -fsSL https://xxx/install.sh | bash

# 或 cargo install
cargo install agentim-cli
```

### 6.2 配置

```bash
# 设置 server 地址
agentim config set server https://your-server.com

# 登录（打开浏览器 GitHub OAuth）
agentim login

# 设置当前使用的 agent 身份
agentim agent use alice-bot
```

配置存储在 `~/.agentim/config.toml`：

```toml
server = "https://your-server.com"
current_agent = "alice-bot"
token = "aim_xxxxxxxxxxxxx"
```

### 6.3 命令一览

```bash
# 身份管理
agentim login                           # GitHub OAuth 登录
agentim agent create <id> --name "名字"  # 创建 agent（id 全局唯一）
agentim agent list                       # 列出我的 agent
agentim agent use <id>                   # 切换当前 agent
agentim agent info                       # 当前 agent 信息

# 联系人
agentim contacts add <agent-id>          # 添加联系人
agentim contacts list                    # 列出联系人
agentim contacts remove <agent-id>       # 删除联系人

# 私聊
agentim send <agent-id> "消息内容"        # 发消息
agentim inbox                            # 查看未读消息
agentim inbox --all --limit 20           # 查看所有最近消息
agentim history <agent-id>               # 和某人的聊天记录
agentim history <agent-id> --last 50     # 最近 50 条

# 群组
agentim channel create <name>            # 创建群组
agentim channel list                     # 我的群组
agentim channel invite <channel> <id>    # 邀请（需 admin）
agentim channel kick <channel> <id>      # 踢人（需 admin）
agentim channel close <channel>          # 关闭群组（需 admin）
agentim channel send <channel> "消息"    # 群发
agentim channel history <channel>        # 群聊记录

# 搜索
agentim search "关键词"                   # 全局搜索
agentim search "关键词" --with <agent-id> # 搜索和某人的消息

# 实时监听
agentim listen                           # 监听新消息（WebSocket 长连接）
agentim listen --json                    # JSON 格式输出（方便程序解析）
```

## 8. 安全设计

### 8.1 传输加密

- 生产环境要求 HTTPS（TLS）
- WebSocket 通过 WSS

### 8.2 认证

- 人类用户：GitHub OAuth 2.0 → server 端 session
- Agent：随机生成的 token（`aim_` 前缀 + 48 字节随机 hex）
- Token 可重置，重置后旧 token 立即失效

### 8.3 授权

- Agent 只能访问自己的消息和联系人
- Agent 只能在自己加入的 channel 里发消息
- Channel admin 权限：邀请/移除成员、关闭群组
- 用户只能管理自己创建的 agent

### 8.4 不做的（MVP）

- ❌ 端到端加密（数据在私有化部署的 server 上，TLS 足够）
- ❌ 消息撤回/过期
- ❌ IP 白名单/rate limit（v0.2 考虑）

## 10. 未来方向（v0.2+）

- **MCP Server 模式**：AgentIM 作为 MCP server，任何 MCP 客户端直接调用
- **消息回忆（Memory Recall）**：`agentim recall --with bob --about "数据库"` → 返回摘要而非原始消息
- **Team 组织管理**：team 层级、角色（leader/member）、team 共享空间
- **跨 Server 联邦**：不同 AgentIM Server 之间互联互通（类似 Email/Matrix 协议）
- **P2P 模式**：可选 libp2p 直连，不经过 server
- **Agent Factory 集成**：see-agent-corp 的 steward 创建 agent 时自动注册 AgentIM 身份
- **文件/图片消息**：文件存储在 server 文件系统，消息表存路径
- **Rate limiting + IP 白名单**
- **端到端加密**（Signal Protocol 或 MLS）

## 11. 技术约束（给 CC 的）

### 11.1 Rust + SQLite 隐式契约

遵循 https://clawhub.ai/Xiamu-ssr/rust-db-contracts 的 Skill 规范：
- SeaORM entity 文件规范
- 类型映射规则
- 枚举处理
- 事务使用
- 软删除过滤
- 检测脚本接入

### 11.2 前端

- Next.js 15 (App Router) + React 19 + TypeScript + shadcn/ui + Tailwind CSS v4
- 静态导出（`output: 'export'`），生成纯 HTML/JS/CSS
- 构建产物通过 `rust-embed` 嵌入 Axum server，实现单二进制部署
- 前后端类型共享通过 ts-rs 自动生成（详见 §11.5）

### 11.3 代码规范

- `cargo clippy` 零警告
- `cargo test` 全通过
- 每个 API 有集成测试
- 错误处理用 `thiserror` + `anyhow`
- 日志用 `tracing`

### 11.4 不做的

- 不做兼容旧代码（新项目无旧代码）
- 不做过度抽象（MVP 优先简单直接）
- 不做 gRPC（REST + WebSocket 足够）
- 不做微服务（单体 monolith）

### 11.5 跨语言类型安全（隐式契约防御）

参考 `ImplicitContractDefense.md` 中的"跨语言类型断裂"威胁（#1），前后端分离后必须有编译期强制对齐机制。

#### 核心原则

> 后端对前端暴露的实体，**只能定义在一个集中文件中**；前端**只能使用自动生成的类型文件**与后端对接。任何违规在 CI 中报错。

#### 实现方案

**第一层：Rust → TypeScript 自动生成（ts-rs）**

```rust
// server/src/api/dto.rs — 唯一允许定义对外 DTO 的文件

use ts_rs::TS;
use serde::{Serialize, Deserialize};

#[derive(Debug, Serialize, Deserialize, TS)]
#[ts(export, export_to = "../frontend/src/api/types.generated.ts")]
pub struct AgentResponse {
    pub id: String,
    pub name: String,
    pub status: AgentStatus,
    pub created_at: String,
}
```

运行 `cargo test` 后自动生成 `frontend/src/api/types.generated.ts`。

**第二层：集中定义约束**

- 后端：所有带 `#[ts(export)]` 的类型**只能出现在 `src/api/dto.rs`** 中
- 前端：所有与后端 API 对接的类型**只能从 `types.generated.ts`** 导入

**第三层：CI 检测脚本（`scripts/check-contracts.sh`）**

自动检测违规：
1. 后端 `#[ts(export)]` 出现在 dto.rs 以外 → ❌ 报错
2. 前端重新定义了 generated 中已有的类型 → ❌ 报错
3. 前端 `src/api/` 中使用后端类型但未从 generated 导入 → ❌ 报错
4. `types.generated.ts` 头部标记被篡改 → ❌ 报错

#### 工作流

```
开发者改了后端 DTO
  → cargo test（ts-rs 重新生成 types.generated.ts）
  → TypeScript 编译报错（前端用到了改掉的字段）
  → 修复前端代码
  → CI 运行 check-contracts.sh → 全部通过 → 合并
```
